## Context

This task extends `WalletService` to support HD wallet creation. It adds the pure-Python Base58Check utilities, the `detect_input_type` routing function, `validate_extended_public_key`, `normalize_to_xpub` (for ypub/zpub API compatibility), and the `_add_hd_wallet` flow including duplicate detection, default tag generation, and background task dispatch.

It also extends the existing `remove_wallet` path to clean up the `hd_address_count:{wallet_id}` config key when an HD wallet is deleted.

## Notes

- `_fetch_initial_hd_data` depends on `RefreshService.refresh_single_hd_wallet` (implemented in T21) and `HistoryService.full_import_hd` (implemented in T22). In this task, stub these methods if needed for the background task call to be syntactically correct ŌĆö tests should mock them.
- The `_b58decode_check` and `_b58encode_check` helpers must not be exported (prefix with `_`). They are implementation details of key validation/conversion only.
- `validate_extended_public_key` returns strings (not exceptions) to mirror `validate_btc_address`. The service raises `AddressValidationError(error_message)`.
- Whitespace trimming happens in `add_wallet()` before routing: `address.strip().replace("\n", "").replace(" ", "")` per ┬¦4.1.b.
- The `XPUB6...` (uppercase) case falls through to `"unknown"` and produces "Invalid Bitcoin address format." ŌĆö this is correct per spec (FR-H03 specifies lowercase prefix is required).
