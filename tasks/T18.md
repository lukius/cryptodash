**Status**: todo

## Context

This task implements the `XpubClient` — a new external API client that queries blockchain.info's `multiaddr` endpoint to retrieve aggregate balance, derived address list, and paginated transactions for a given xpub key. It follows the same `BaseClient` pattern as the existing `BitcoinClient` and `KaspaClient`.

This client is the sole data source for all HD wallet balance and history operations. It must be self-contained and unit-testable with mocked HTTP responses.

## Notes

- Inherit from `backend/clients/base.py`'s `BaseClient` exactly as `BitcoinClient` does — use `self._get_with_retry(path, params)` for all HTTP calls.
- The `n=1` parameter on the summary call is intentional: it minimizes the `txs` array in the response while still returning the full `addresses` array (which is not paginated by `n`/`offset`).
- The 0.2s sleep in `get_xpub_transactions_all` respects blockchain.info's undocumented rate limit. Use `asyncio.sleep(0.2)`.
- Tests must mock the HTTP layer (use `httpx` mock or `respx`), not make real network calls.
