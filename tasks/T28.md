# T28: Fix BUG-02 — WalletDetailView uses wrong truncation call for HD wallet address display

**Status**: todo
**Layer**: frontend
**Assignee**: developer
**Depends on**: none

## Context

`WalletDetailView.vue` line 197 calls `truncateAddress(wallet.address, 12, 8)`, which shows the first 12 characters and last 8 characters of the xpub key. FR-H09 requires the first 10 characters and last 6 characters ("xpub6CUGRo...d4e7f2"). The utility function `formatWalletAddress(address, walletType)` from `@/utils/format` already implements FR-H09 correctly — it is not being used here.

> "The system shall display the extended public key in the address field, truncated to the first 10 characters + '...' + last 6 characters."
> — FUNC_SPEC_HD_WALLETS.md FR-H09

> "Truncation for display is performed client-side in the frontend utility format.ts."
> — TECH_SPEC_HD_WALLETS.md §4.10.a

## Files owned

- `frontend/src/views/WalletDetailView.vue` (modify — line 197: replace `truncateAddress(wallet.address, 12, 8)` with `formatWalletAddress(wallet.address, wallet.wallet_type)`)

## Subtasks

- [ ] ST1: On line 197 of `frontend/src/views/WalletDetailView.vue`, replace `truncateAddress(wallet.address, 12, 8)` with `formatWalletAddress(wallet.address, wallet.wallet_type)`.
- [ ] ST2: Confirm `formatWalletAddress` is already imported in `WalletDetailView.vue`. If not, add the import: `import { formatWalletAddress } from '@/utils/format'`.
- [ ] ST3: Confirm `truncateAddress` is not used anywhere else in `WalletDetailView.vue`. If it is no longer used at all in this file after the fix, remove its import to avoid a lint warning.
