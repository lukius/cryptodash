# T29: Fix BUG-03 — Delete confirmation dialog body text not HD-aware

**Status**: todo
**Layer**: frontend
**Assignee**: developer
**Depends on**: none

## Context

`RemoveWalletDialog.vue` line 69 renders the dialog body as:

```
"All historical data for this wallet will be deleted."
```

For HD wallets, the functional spec requires the message:

> "Remove '{tag}'? All historical data for this HD wallet will be deleted."
> — FUNC_SPEC_HD_WALLETS.md §5.1.c (Removing an HD Wallet)

The dialog title already shows `Remove '{{ wallet.tag }}'?` correctly. Only the body paragraph needs to be HD-aware.

## Files owned

- `frontend/src/components/wallet/RemoveWalletDialog.vue` (modify — line 69: make body text conditional on `wallet.wallet_type`)

## Subtasks

- [ ] ST1: On line 69 of `frontend/src/components/wallet/RemoveWalletDialog.vue`, replace the static `<p>` with a conditional:
  ```vue
  <p v-if="wallet.wallet_type === 'hd'">
    All historical data for this HD wallet will be deleted.
  </p>
  <p v-else>
    All historical data for this wallet will be deleted.
  </p>
  ```
- [ ] ST2: Confirm `wallet.wallet_type` is available in the component's props or the wallet object passed to the dialog. If the component receives the full `WalletResponse` object as a prop, `wallet_type` is already present (added by T23/T24). If it only receives `wallet.tag`, extend the prop to include `wallet_type`.
