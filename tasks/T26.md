## Notes

- The `expandedWalletId` state in `WalletTable.vue` is local component state (`ref`), not in the Pinia store. Only one HD wallet row is expanded at a time (clicking another row collapses the previous one).
- On `WalletDetailView.vue`, the `DerivedAddressList` defaults to expanded with no toggle — the user navigated specifically to this wallet's page, so showing the full address list immediately is the right UX.
- The error prop for `DerivedAddressList` in `WalletTable.vue` should be computed as: `!wallet.hd_loading && wallet.derived_addresses === null && wallet.warning !== null` — per §7.3 template snippet.
- Verify that `frontend/src/stores/wallets.ts` stores the full `WalletResponse` shape including HD fields. If the store transforms API responses before storing them, ensure no HD fields are dropped.
- The `<Transition name="expand">` requires a corresponding CSS rule for `expand-enter-active` / `expand-leave-active` transitions. Add these to `WalletTable.vue`'s `<style>` section or to the global stylesheet if one exists.
